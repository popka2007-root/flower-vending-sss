# Платформа управления цветочным вендинговым аппаратом (Flower Vending System)

**Flower Vending System** — это передовая платформа управления киоском по продаже премиальных букетов цветов. Система разработана в режиме "первичности симулятора" (Simulator-first), что позволяет проводить глубокое тестирование логики без необходимости реального оборудования, но при этом она полностью готова к работе с реальным "железом" в режиме киоска (kiosk mode) через сенсорный экран.

Система взаимодействует с купюроприёмником, моторами выдачи товара, обеспечивает контроль температуры и включает продвинутые механизмы обработки сбоев.

---

## 1. Ключевые возможности

*   **Интерактивный каталог товаров:** Удобный интерфейс выбора букетов, корзина и оформление заказа через сенсорный экран.
*   **Приём наличных:** Интеграция с купюроприёмником (JCM DBV-300-SD) с проверкой возможности выдачи сдачи.
*   **Управление выдачей:** Контроль моторных слотов и дверец выдачи товара.
*   **Климат-контроль:** Мониторинг и контроль температуры внутри витрины для поддержания свежести цветов.
*   **Отказоустойчивость (Crash recovery):** Журналирование намерений и результатов (intent/outcome) для каждого аппаратного действия. При перезапуске система восстанавливает прерванные транзакции.
*   **Сервисный режим:** Защищенный PIN-кодом доступ для операторов (дашборд, диагностика, управление инвентарем).
*   **Интеграции:** Клиент для обмена данными с 1С и публикация телеметрии.

---

## 2. Стек технологий

Проект построен с использованием современных и надежных технологий, придерживаясь строгих архитектурных стандартов:

*   **Язык:** Python 3.11+
*   **Пользовательский интерфейс (UI):** PySide6 (Qt6)
*   **Асинхронность:** `asyncio` для эффективной работы с устройствами и сетью.
*   **Конфигурация:** Pydantic v2 + PyYAML (надежная типизация настроек).
*   **База данных:** SQLite (с режимами WAL + FULL sync для предотвращения потери данных).
*   **Взаимодействие с оборудованием:** `pyserial` (UART), интеграция с ESP32 Arduino (C++) для управления моторами/сенсорами.
*   **Сборка и упаковка:** Hatchling, PyInstaller, Inno Setup, AppImage.
*   **Качество кода:** Ruff (линтер и форматтер), mypy (строгая типизация), pre-commit.
*   **Тестирование:** pytest, pytest-asyncio (более 100 тестов: unit, integration, e2e).

## 3. Архитектура проекта

Проект использует строгое разделение по принципам **Clean Architecture** (Чистая Архитектура), где зависимости направлены исключительно внутрь, к доменному ядру.

```text
UI LAYER (PySide6)
       ↓ (async вызовы)
APPLICATION FACADE (Мост между UI и ядром)
       ↓
APPLICATION CORE (Оркестраторы, Конечный автомат, Шина команд и событий)
       ↓
DOMAIN CORE (Сущности, Бизнес-логика — НИКАКИХ внешних зависимостей)
       ↑
INFRASTRUCTURE (БД SQLite, Журналирование, Конфигурация) & DEVICES (Адаптеры оборудования)
```

*   **Domain Core:** Сердце системы. Здесь нет импортов фреймворков или библиотек (кроме стандартных). Описывает бизнес-правила.
*   **Application Core:** Конечный автомат (FSM) из 21 состояния, который строго контролирует процесс продажи (от приветствия до выдачи сдачи и товара).
*   **Инфраструктура и Устройства:** Реализуют интерфейсы, определенные в ядре. Включают симуляторы для разработки без реального "железа".

## 4. Быстрый старт (Разработка)

### Установка зависимостей
```bash
pip install -r requirements.txt -r requirements-dev.txt -r requirements-ui.txt
# Или с помощью Make:
make install-all
```

### Запуск симулятора с UI
Это запустит приложение с виртуальными устройствами, что идеально подходит для тестирования логики.
```bash
python -m flower_vending simulator-ui --config config/examples/machine.simulator.yaml
```

### Дополнительные команды
*   **Диагностика:** `python -m flower_vending diagnostics`
*   **Проверка конфигурации:** `python -m flower_vending validate-config`
*   **Запуск тестов:** `make test` или `make test-e2e`

## 5. Планы по развитию и улучшению (Roadmap)

Для дальнейшего развития проекта и улучшения пользовательского и административного опыта запланированы следующие шаги:

1.  **Доработка аппаратной части (Hardware):**
    *   Завершение интеграции и отладки работы с контроллерами, моторами выдачи, датчиками и купюроприемником на реальном оборудовании.
    *   Повышение стабильности работы периферии в условиях нестабильного электропитания или потери связи с USB-портами.
2.  **Обновление дизайна главной страницы:**
    *   Пересмотр цветовой гаммы интерфейса для повышения привлекательности витрины.
    *   Улучшение UX/UI (анимации, удобство касаний).
3.  **Улучшение панели администратора (Сервисный режим):**
    *   Доработка дашборда оператора для более удобной инкассации, загрузки товаров и просмотра диагностики прямо на экране автомата.
4.  **Оптимизация производительности на слабом "железе":**
    *   Проверка плавности анимаций PySide6 на целевых промышленных компьютерах и минимизация потребления ресурсов.
