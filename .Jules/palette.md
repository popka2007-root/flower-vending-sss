## 2024-05-27 - Refactored raw QPushButtons to IconButton
**Learning:** Found multiple instances where standalone icon-only `QPushButton` widgets were manually configured with `.setIcon`, `.setFixedSize`, and custom stylesheets that hid hover states (or lacked them entirely). The `modern.py` widgets module already contains a reusable `IconButton` class that handles this cleanly. Replacing these adds automatic hover states (visual feedback), reduces boilerplate, and makes the UI more consistent. Crucially, the original buttons *did* have `setAccessibleName` configured properly, which is good practice for PySide6 a11y, and this was preserved during refactoring.
**Action:** Always check `modern.py` for reusable UI components before creating raw `QPushButton` configurations. When refactoring interactive elements, pay close attention to preserving existing `setAccessibleName` attributes to maintain accessibility while improving the visual code structure.
